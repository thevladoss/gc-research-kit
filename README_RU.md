# GCHAR — кит для исследования «Великой борьбы» в Claude Code

## Что внутри

```
gc-research-kit/
├── CLAUDE.md                      # мозг проекта: пайплайн, правила, схема вердиктов, метрики
├── STARTER_PROMPT.md              # первый промпт для Claude Code (скопировать целиком)
├── schemas/claim.schema.json      # контракт данных между всеми стадиями
├── .claude/
│   ├── agents/
│   │   ├── claim-extractor.md     # стадия 1: нейтральное извлечение утверждений
│   │   ├── claim-verifier.md      # стадия 2: верификация веб-поиском (+ проверка анахронизмов)
│   │   ├── defender.md            # стадия 3: адверсариальная защита (steelman)
│   │   ├── impact-assessor.md     # стадия 4: severity × centrality, load-bearing
│   │   └── synthesizer.md         # стадия 5: итоговый синтез (на русском)
│   └── commands/
│       └── process-chapter.md     # /process-chapter N — оркестрация одной главы
└── README_RU.md                   # этот файл
```

## Запуск за 4 шага

1. Скопируй папку целиком как корень проекта (например `~/projects/gc-research/`). Важно: `.claude/` — скрытая папка, проверь что она скопировалась (`ls -la`).
2. Открой терминал в этой папке и запусти `claude`. Субагенты подгружаются при старте сессии — раз они уже на диске, всё подхватится сразу. Проверить можно командой `/agents` (должны быть видны все 5).
3. Вставь первым сообщением содержимое `STARTER_PROMPT.md` (всё до раздела "After calibration").
4. После калибровки на главах 4–6 внимательно посмотри отчёт, поправь что нужно, и только потом одобряй батч.

## Модели

Во всех агентах в frontmatter стоит `model: claude-fable-5` — стадии verify/defend/impact/synthesis требуют максимума рассуждений, а у тебя есть время и лимиты. Если строка модели не подхватится (Claude Code иногда хочет alias), запусти `/agents`, открой агента и выбери модель из списка, либо спроси у самого Claude Code актуальное имя. Если лимиты начнут поджимать — extractor можно опустить до Sonnet 4.6, это наименее рискованная экономия; verifier и defender не понижай, там качество решает всё.

## Что можно (не обязательно) подготовить заранее в scholarship/

- Donald McAdams — исследование исторических глав ВБ (эталон по главе о Гусе)

Пайплайн работает и без них — они лишь ориентируют поиск, вердикты всегда выносятся по независимым источникам (это зашито в правила агентов).

## Практические советы

- **Калибровка — святое.** Первые 3 главы покажут, не слишком ли мелко/крупно режет экстрактор и не перегибает ли верификатор. Дешевле поправить промпт агента после 3 глав, чем после 43.
- Правки файлов агентов вступают в силу после перезапуска сессии (`/agents`-редактор применяет сразу).
- Коммить после каждой главы (зашито в стартовый промпт) — это твоя страховка и лог прогресса.
- Полный прогон — это десятки часов работы агентов и много веб-поиска. Гони батчами по 5–10 глав, между батчами проглядывай сводки.
- Здоровые ожидания по данным: заметная доля вердиктов должна быть `anachronistic` (ошибки, унаследованные от Уайли/Д'Обинье), а defender должен выигрывать часть споров. Если contradicted ≈ 100% и защита никогда не срабатывает — пайплайн перекошен, стоит ужесточить верификатора.
